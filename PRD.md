### 产品需求文档 (PRD) - DCleaner

**文档版本:** V1.1
**产品名称:** DCleaner
**产品定位:** 一款轻量、无广告、纯净的 Windows 系统清理与注册表优化工具。
**核心价值:** 替代日益臃肿的传统清理软件，回归“清理”本质。通过 Tauri 架构实现极低的资源占用和现代化的流畅 UI，利用 Rust 提供安全、高效的底层系统调用，并将清理控制权完全交还给用户。

---

#### 1. 核心交互流程 (Core Interaction Flow)

为了保障用户数据安全与知情权，所有清理与修复动作必须遵循以下核心流程：

1. **深度扫描 (Scan):** 用户点击扫描后，Rust 后端执行只读操作，遍历目标路径或注册表项。
2. **结果展示 (Display):** 扫描完成后，前端以结构化列表呈现即将清理的数据详情（分类、路径、大小、风险等级），默认勾选安全项，高风险项需用户手动勾选。
3. **冲突检测与提示 (Prompt):** 若勾选的清理项涉及正在运行的进程（如浏览器缓存），系统需弹窗提示用户该进程正在运行，并提供“跳过”或“强制结束进程并继续”的选项。
4. **执行清理 (Execute):** 用户最终确认后，后端才执行真实的删除或修改操作，并实时反馈进度。

---

#### 2. 技术架构要求

* **应用框架:** Tauri
* **核心逻辑端 (Backend):** Rust。负责所有高权限的系统级操作，包括文件系统 I/O（扫描/删除临时文件）、进程查询与查杀（检测进程冲突并执行结束指令）以及调用 Windows API（操作注册表）。
* **用户界面端 (Frontend):** 推荐使用轻量级前端框架（如 Vue 3 或原生 Web Component），通过 Tauri IPC 与 Rust 后端进行异步通信，确保 UI 在深度扫描和等待用户确认时保持响应。

---

#### 3. 核心功能需求 (Functional Requirements)

**3.1 Windows 系统清理模块**

* **功能描述:** 扫描并安全移除 Windows 系统运行过程中产生的冗余文件。
* **具体清理项:** 系统临时文件、回收站、剪贴板历史、内存转储文件、Windows 日志文件、DNS 缓存、系统预读文件。
* **交互要求:** 扫描结果需明确显示各类别占用的磁盘空间。对于回收站内容，允许展开查看具体文件列表。

**3.2 浏览器清理模块**

* **功能描述:** 检测系统中安装的主流浏览器，清理隐私数据与缓存。
* **支持的浏览器:** Chromium 内核系列 (Chrome, Edge 等) 和 Firefox。
* **具体清理项:** 缓存 (Cache)、历史记录 (History)、Cookie (支持白名单排除)、下载历史、Session 数据。
* **进程冲突检测逻辑:** * 在用户点击“执行清理”时，检查目标浏览器进程（如 `chrome.exe`, `msedge.exe`）是否存活。
* **若存活，强制弹窗提示:** “检测到 [浏览器名称] 正在运行，清理需要关闭该程序。是否立即结束该进程？” -> 选项：[结束进程并清理] / [跳过此项]。



**3.3 第三方应用程序清理模块**

* **功能描述:** 清理常用第三方软件产生的日志、缓存和历史记录。
* **支持范围:** 开发与效率工具、多媒体播放器、解压缩软件等。
* **进程冲突检测逻辑:** 与浏览器清理模块一致，若目标应用的进程正在运行，必须先弹窗请求杀进程许可。

**3.4 注册表扫描与清理模块**

* **功能描述:** 深度扫描 Windows 注册表，定位无用或错误的键值，提供预览后进行清理。
* **具体扫描项:** * 缺失的共享 DLL
* 未使用的文件扩展名
* 卸载软件残留路径
* 无效的应用路径
* 无效的字体记录
* 失效的开机启动项


* **交互要求:** 扫描结果必须清晰列出每个异常键值的具体路径和“修复建议”。

**3.5 注册表备份与恢复机制 (安全基石)**

* **功能描述:** 注册表操作极具风险，必须提供防呆与撤销机制。
* **执行逻辑:**
* 用户点击“修复/清理”注册表并确认列表后，**强制执行备份动作**。
* 将即将被删除的节点完整导出至本地 `.reg` 文件（默认保存在应用的 `Backup` 文件夹，按时间戳命名）。
* 提供独立的“恢复中心”入口，允许用户查看历史清理记录，并通过双击或一键执行 `.reg` 文件撤销操作。



---

#### 4. 非功能需求 (Non-Functional Requirements)

* **性能要求:** 安装包体积控制在 10MB 左右；扫描与清理采用多线程/异步处理；IPC 通信数据量大时需进行分页或批处理，防止前端渲染卡顿。
* **安全与权限:** 启动时或执行核心清理时，需通过 UAC (User Account Control) 获取管理员权限。
* **UI/UX 设计:** * 绝对的无打扰，无广告、无驻留、无捆绑。
* 清晰的“扫描 -> 确认 -> 提示 -> 执行”视觉引导流程。
* 提供“一键全选/取消全选”以及针对单个文件的精准勾选功能。
